# CLAUDE.md — Rugby ESP32 Display

Guidance for Claude Code when working on this project.

## Project

Live rugby scoreboard on **Adafruit MatrixPortal S3** driving **2× chained HUB75 128×64 panels → 256×64 pixels**.
Displays results, live scores, standings, and fixtures for Top 14, Pro D2, and Champions Cup.

Full spec: `prompt_ESP.md`. Design doc: `docs/superpowers/specs/2026-04-14-rugby-esp32-design.md`.

---

## Hardware

| | |
|--|--|
| Board | Adafruit MatrixPortal S3 (ESP32-S3, 8MB Flash, 320KB SRAM, **2MB PSRAM**) |
| Display | 2× HUB75 128×64 chained → **256×64 total** |
| Framework | Arduino on PlatformIO |

**PSRAM enabled** (`BOARD_HAS_PSRAM`, `qio_qspi`). Logo cache (~350KB) and DMA framebuffer live in PSRAM. SRAM heap is tight (~90KB after boot).

---

## Build

```bash
pio run -e matrixportal_s3              # build
pio run -e matrixportal_s3 --target upload     # firmware
pio run -e matrixportal_s3 --target uploadfs   # LittleFS (logos + web UI)
python3 tools/convert_logos.py          # regenerate logos → data/logos/*.bin
```

Key `platformio.ini` flags:
- `-DBOARD_HAS_PSRAM`, `board_build.arduino.memory_type = qio_qspi`
- FreeRTOS task stacks: bootFetch 12KB, DataFetcher 12KB, Renderer 4KB

## Architecture

### FreeRTOS Tasks

| Core | Task | Stack | Notes |
|------|------|-------|-------|
| Core 0 | `BootFetch` | 12KB | One-shot: WiFi + NTP + initial fetch. Deleted after. |
| Core 0 | `DataFetcher` | 12KB | Periodic polling (live/normal), NTP sync. |
| Core 1 | `Renderer` | 4KB | 30fps `Scenes.tick()` + `Display.flip()`. |

HUB75 DMA is interrupt-sensitive — rendering stays on Core 1. No network on Core 1.

### Memory

- **PSRAM (2MB):** Logo cache (84 logos, ~350KB), DMA framebuffer
- **SRAM heap:** ArduinoJson, AsyncWebServer, WiFi/TLS buffers, task stacks
- **LittleFS (~4MB):** logos, `index.html`, `/cache.json`

### Data flow

```
Idalgo HTML (stream, 4KB chunks)
  → IdalgoParser → CompetitionData → MatchDB (mutex-protected)
  → SceneManager::rebuildSlots() → scenes render from MatchDB snapshot
```

## Visual Style

- **Font:** Atkinson Hyperlegible (8pt, 10pt, 12pt bold)
- **Text shadow:** grey offset (+1,+1 or +2,+2) behind main text
- **Colors:** winner=gold `C_GOLD`, loser=white, draw=white, live=green `C_GREEN`
- Accents stripped for display via `TeamData.h::stripAccents()`

### Scene Layouts (256×64)

**ScoreboardScene**
- Team logos 64×64 left/right
- Competition logo centered top (variable width, height=48)
- Round indicator `J26` bottom-left of home logo
- Scores in bold 12pt beside comp logo, with grey relief shadow
- Status: "Final" / "MT" / "45'" / "Live" at bottom center
- Counter `%d/%d` bottom-right

**FixturesScene**
- Same logo layout as scoreboard
- Round indicator `J28` bottom-left of home logo
- Team abbrevs at y=36 (was 44), pushed to edges
- Date at y=48 (was 54), time at y=63 (if not TBD)
- Counter bottom-right

**StandingsScene**
- Full-height scroll (no header bar, no "CLASSEMENT" text)
- Competition logo overlaid on right side (48px height, ~40px from edge)
- Rows: mini logo 16×16 + rank + name + points
- Scroll: 1s pause at start, scroll through all rows, 0.5s pause at end
- Scroll speed calibrated to scene duration (`DisplayPrefs.standing_s`, default 20s)
- Zone colors: playoffs→gold, relegation→red

## Data Sources

**Idalgo (ladepeche.fr)** — sole source for results, fixtures, standings.

- Stream-parsed in 8KB chunks (cannot hold full ~500KB HTML in SRAM)
- Headers: `User-Agent: Mozilla/5.0`, `Accept-Language: fr-FR`, `Accept-Encoding: identity`
- `data-status`: `"0"`=scheduled, `"1"`/`"3"`=finished, `"2"`=treat as finished
- Match blocks without `data-status` default to Scheduled; infer Finished only if scores present
- **Dates:** found in parent `div_idalgo_content_result_date_list_title > span` (e.g. "Samedi 18 avril 2026"), not in match block itself. Parser scans for these titles before match blocks and stores `_lastDateStr`.
- **Round detection:** reads `span_idalgo_content_competition_navigation_days_listbox_current` for the active journee (not max from dropdown)
- **Round IDs:** `round_ids[40]` stores exact `round → url_id` mapping parsed from `/resultats/ID/journee-N` links. Quick scan before full parsing prevents losing links when buffer drops.
- **URL math:** `fetchNextJournee` uses `round_ids[round]` if available, else falls back to `round_url_base - round`

## WiFi & Persistence

- **Credentials:** stored in **Preferences (NVS)** namespace `"wificfg"` — survives `uploadfs`
- **Display prefs:** also in Preferences namespace `"rugby"` (brightness, enabled comps, durations)
- Fallback to compile-time `WIFI_SSID` / `WIFI_PASSWORD` from `include/credentials.h`
- Web UI endpoints: `/status`, `/config` (brightness), `/prefs` (scenes), `/wifi` (networks), `/restart`, `/next-scene`

## Key Implementation Notes

### Timezone
```cpp
configTime(0, 0, "time.cloudflare.com", "162.159.200.1", "216.239.35.0");
setenv("TZ", "CET-1CEST,M3.5.0,M10.5.0/3", 1);
tzset();
```
Store kickoffs as UTC epoch. Convert to Paris local only at display with `localtime()`.

### Thread safety
- `MatchDB` protected by FreeRTOS mutex. Writers (Core 0) take it fully. Readers (Core 1) use `acquire*/release()`.
- `SceneManager::requestRebuild()` + `markDirty()` are thread-safe flags checked in `tick()`.
- `Display.flip()` must only be called from renderer task (Core 1).

### SRAM heap pressure
After boot: ~90KB free. After web services start: ~40–50KB.
- `AsyncWebServer` + mDNS + OTA consume ~20–30KB
- `WiFiClientSecure` TLS buffers are large and may not fully free
- Mitigations: reduced task stacks, defer web services if heap < 50KB, parser buffer at 8KB
